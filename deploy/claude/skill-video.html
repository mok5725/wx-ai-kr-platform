claude-video

클로드가 영상을 "보게" 하기

클로드는 웹페이지·코드는 읽어도 영상은 기본으로 못 봅니다. 이 /watch 스킬은 영상의 화면(프레임)과 소리(자막·전사)를 뽑아 클로드에게 넘겨, 실제 내용을 근거로 답하게 합니다.

🎯 이 장에서 배우는 것
영상 분석 파이프라인의 원리·설치(특히 Windows)·명령·주의.

한눈에

무엇인가

영상을 다운로드·프레임 추출·전사해 클로드가 분석하게 하는 스킬

핵심 원리

자막 우선 → 필요 시 다운로드 → 장면변화 프레임 + Whisper 전사

설치 난이도

보통 (Windows는 ffmpeg·yt-dlp 수동 설치)

사용자 추천도 (참고)

★★★☆☆ — 강력하지만 준비물·비용 있음

작동 원리 (자세히)

자막 있으면 자막만(빠름)→ 없으면 영상 다운로드→ 장면변화 프레임 추출→ 비슷한 프레임 제거→ Whisper 전사

프레임은 고정 간격이 아니라 장면이 바뀌는 지점을 골라 뽑고, 거의 같은 프레임은 버려 토큰을 아낍니다(자막이 있으면 다운로드·이미지 토큰 0). 프레임에는 MM:SS 타임스탬프가 붙어, "30초에 뭐가?" 같은 질문에 정확히 답할 수 있습니다.

상세 모드(--detail)

transcript(자막만·0토큰) · efficient(키프레임·빠름) · balanced(기본·장면변화) · token-burner(최대 커버리지). 전사는 Groq(기본)·OpenAI Whisper를 쓰며 자막이 없을 때만 필요합니다.

설치

Claude Code
/plugin marketplace add bradautomates/claude-video
/plugin install watch@claude-video
⚠️ 전제조건 · Windows 주의: ffmpeg와 yt-dlp가 필요합니다. macOS는 첫 실행 때 자동 설치되지만, Windows는 자동 설치가 안 되고 안내(winget/pip)만 나오므로 직접 설치해 PATH에 등록해야 합니다. Whisper API 키(Groq/OpenAI)는 자막 없는 영상에만 필요(선택).

참고: npx skills add bradautomates/claude-video -g 로 다른 호스트에도 설치 가능. claude.ai 웹은 릴리스의 watch.skill 업로드.

사용법

형식: /watch <URL 또는 경로> [질문] [옵션]

옵션의미
--detail [모드]화질/속도 조절(transcript~token-burner)
--start / --end특정 구간만(예: 2:15~2:45) — 촘촘·저비용
--timestamps특정 시점 프레임 콕 집기
--resolution W프레임 가로폭(기본 512, 글자 읽으려면 1024)
--no-whisper / --no-dedup전사 끄기 / 중복 프레임 유지
예시
> /watch https://youtu.be/영상ID  30초쯤 무슨 일이 일어나?
> /watch ~/영상/화면녹화.mp4  UI가 언제 깨져?
> /watch "URL" --start 2:15 --end 2:45      (구간 집중)
> /watch video.mp4 --detail token-burner --resolution 1024

장점 · 단점

👍 장점

영상의 실제 내용을 질문 가능. 자막 우선·장면변화·중복 제거로 토큰 효율. 타임스탬프로 정확한 참조. 폭넓은 소스·로컬 파일 지원.

👎 단점

Windows 설치 마찰(ffmpeg·yt-dlp 수동). 자막 없는 영상은 유료 Whisper API 비용. 다운로드·처리 시간. 긴 영상은 커버리지 얇아짐(구간 지정 권장). 플랫폼 변경·로그인 게이트로 다운로드 실패 가능.

when

언제 쓰고, 언제 피하나

쓰기 좋을 때: 긴 영상 요약, 영상 구조·훅 분석, 화면 녹화로 버그 진단, 튜토리얼의 특정 순간 추출.
피할 때: 이미 좋은 자막을 붙여넣을 수 있을 때(더 쌈), 로그인 게이트·미지원 소스, ffmpeg·yt-dlp를 깔 수 없는 Windows, 매우 긴 영상을 구간 지정 없이 볼 때(비용).
