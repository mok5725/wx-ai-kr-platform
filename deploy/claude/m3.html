핵심 워크플로우

탐색 → 계획 → 구현

클로드 코드를 잘 쓰는 사람의 비결은 딱 하나입니다. 바로 만들지 말고, 계획을 먼저 받아 검토하는 것. 이 습관이 결과 품질을 좌우합니다.

🎯 이 단계를 마치면
계획을 한 번 수정한 뒤 산출물을 완성할 수 있습니다.

1. 좋은 작업의 3박자

① 탐색

지금 폴더에 뭐가 있고 무엇이 필요한지 파악.

→
② 계획

무엇을 어떤 순서로 만들지 글로 정리.

→
③ 구현

승인된 계획대로 실제 결과물을 제작.

가장 흔한 실수는 ②를 건너뛰는 것입니다. 계획을 먼저 보면, 엉뚱한 결과에 시간을 버리지 않습니다.

2. Plan 모드 — "만들기 전에 계획부터"

클로드 코드에는 바로 파일을 고치지 않고 계획만 먼저 세워 보여주는 방식이 있습니다. 계획을 보고 "좋아, 이대로 해줘"라고 승인하면 그때 실제 작업이 시작됩니다.

Claude Code 
> 바로 만들지 말고, 먼저 계획을 세워서 보여줘.
  내가 확인하고 승인하면 그때 진행해줘.

▸ 이렇게 요청하면 클로드가 단계별 계획을 먼저 제시합니다. Shift+Tab을 눌러 계획(Plan) 모드로 바꾸면, 클로드가 읽고 계획만 세우고 파일은 고치지 않습니다 — 계획을 승인해야 실제 작업이 시작됩니다.

3. 계획을 검토·수정하기 (진짜 실력)

계획을 그냥 통과시키지 말고, 빠진 것·바꿀 것을 짚어 수정하게 하세요. 이게 클로드 코드 활용의 핵심 역량입니다.

Claude Code 
> 계획 좋아. 그런데 2번에 '수료번호 자동 생성'을 추가하고,
  전체를 모바일에서도 보이게 반응형으로 바꿔줘. 그다음 진행해.

4. 대표 실습 — 수료증 생성기 (따라하기 · 더미데이터)

배운 3박자(탐색→계획→구현)를 그대로 적용해 봅니다. 아래 순서대로 따라오세요. (연습용 데이터: 실습자료/수료자명단.csv)

① 요청 — 바로 만들지 말고 계획부터

Claude Code
> 교육 수료증을 만드는 간단한 웹페이지를 만들고 싶어.
  이름과 과정명을 입력하면 수료증 미리보기가 나오게.
  바로 만들지 말고 계획부터 세워서 보여줘. (연습용 가짜 데이터로)

② 클로드가 이런 계획을 보여줍니다 (예시)

1. 이름·과정명·수료일 입력 폼 만들기 → 확인: 화면에 폼이 보이는가
2. 입력을 즉시 반영하는 수료증 미리보기 카드 → 확인: 글자를 치면 바로 바뀌는가
3. 인쇄·저장 버튼 추가 → 확인: 미리보기가 인쇄 형식으로 나오는가

③ 계획을 한 번 수정 (핵심 역량)

④ 완성 & 확인

승인하면 클로드가 실제 파일(예: certificate.html)을 만듭니다. 브라우저로 열어 이름·과정명을 넣어 보고, 결과가 요청대로인지 눈으로 확인하세요(다음 7단계 검증 습관). 어긋난 점은 다시 말해 고치면 됩니다.
